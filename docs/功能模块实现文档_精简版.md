1）用户认证模块实现

（1）项目前端实现相关代码

**`frontend/js/auth.js`** — 全局认证工具对象，管理 Token 与用户信息的本地存储。

```javascript
const Auth = {
  save(token, user) {
    localStorage.setItem('mq_token', token);
    localStorage.setItem('mq_user', JSON.stringify(user));
  },
  getToken() { return localStorage.getItem('mq_token'); },
  getUser()  { const u = localStorage.getItem('mq_user'); return u ? JSON.parse(u) : null; },
  isLoggedIn() { return !!this.getToken(); },
  isAdmin()    { const u = this.getUser(); return u && u.role === 'ADMIN'; },
  logout()     { localStorage.removeItem('mq_token'); localStorage.removeItem('mq_user'); window.location.href = '/login.html'; },
  requireLogin()  { if (!this.isLoggedIn()) window.location.href = '/login.html'; },
  requireAdmin()  { if (!this.isAdmin()) window.location.href = '/index.html'; }
};
```

**`frontend/login.html`** — 登录表单，调用 API 后保存登录态并按角色跳转。

```javascript
async function login() {
  const username = document.getElementById('username').value.trim();
  const password = document.getElementById('password').value;
  if (!username || !password) return showToast('请填写用户名和密码', 'error');
  const data = await api.login(username, password);
  Auth.save(data.token, { nickname: data.nickname, role: data.role });
  window.location.href = data.role === 'ADMIN' ? '/admin/index.html' : '/index.html';
}
```

**`frontend/register.html`** — 注册表单，含格式校验与密码可见性切换。

```javascript
async function register() {
  // 前端校验：非空、长度、密码一致、邮箱格式
  if (username.length < 3) return showToast('用户名至少3个字符', 'error');
  if (password.length < 6) return showToast('密码至少6位', 'error');
  if (password !== confirmPassword) return showToast('两次密码不一致', 'error');
  if (!/^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(email)) return showToast('邮箱格式不正确', 'error');
  await api.register({ username, password, confirmPassword, nickname, email });
  window.location.href = '/login.html';
}
```

（2）项目后端实现相关代码

**`AuthController`**（`controller/AuthController.java`）— 认证接口，路径前缀 `/api/auth`，无需 Token。

```java
@RestController
@RequestMapping("/api/auth")
public class AuthController {
    @PostMapping("/register")
    public R<?> register(@Valid @RequestBody RegisterRequest req) {
        authService.register(req);
        return R.ok("注册成功");
    }

    @PostMapping("/login")
    public R<?> login(@Valid @RequestBody LoginRequest req) {
        return R.ok(authService.login(req)); // 返回 { token, role, nickname }
    }

    @GetMapping("/me")
    public R<User> me(@AuthenticationPrincipal UserDetails userDetails) {
        User user = authService.getCurrentUser(userDetails.getUsername());
        user.setPasswordHash(null); // 脱敏处理
        return R.ok(user);
    }
}
```

**`AuthService`**（`service/AuthService.java`）— 注册校验唯一性、加密密码；登录验证密码并生成 JWT。

```java
public void register(RegisterRequest req) {
    if (userMapper.selectOne(...eq("username", req.getUsername())) != null)
        throw new RuntimeException("用户名已存在");
    user.setPasswordHash(passwordEncoder.encode(req.getPassword())); // MD5 加密
    user.setRole("USER"); user.setIsActive(1);
    userMapper.insert(user);
}

public Map<String, String> login(LoginRequest req) {
    if (user == null || !passwordEncoder.matches(req.getPassword(), user.getPasswordHash()))
        throw new RuntimeException("用户名或密码错误");
    String token = tokenProvider.generateToken(user.getUsername(), user.getRole());
    return Map.of("token", token, "role", user.getRole(), "nickname", user.getNickname());
}
```

**`JwtTokenProvider`**（`security/JwtTokenProvider.java`）— 生成、验证 JWT，从 Payload 提取用户名和角色。

**`JwtAuthFilter`**（`security/JwtAuthFilter.java`）— 过滤器，解析请求头中的 Bearer Token，写入 SecurityContext。

**`SecurityConfig`**（`config/SecurityConfig.java`）— 权限规则：注册/登录公开，管理接口限 ADMIN，其他需登录。

```javascript
uploadImage: async (file) => {
  const fd = new FormData();
  fd.append('file', file); // 字段名须与后端 @RequestParam("file") 一致
  // 注意：不手动设置 Content-Type，让浏览器自动加 multipart boundary
  const res = await fetch(BASE + '/upload/image', { method: 'POST', headers, body: fd });
  return data.data.url; // 返回 /uploads/xxxxxxxx.jpg

```

（2）项目后端实现相关代码

**`FileController`**（`controller/FileController.java`）— 接收 multipart 文件，调用 `FileService` 保存后返回访问 URL。

**`FileService`**（`service/FileService.java`）— 生成 UUID 文件名防冲突，自动创建上传目录，写入文件流，返回前端可访问的相对路径。

```java
public String save(MultipartFile file) throws IOException {
    String filename = UUID.randomUUID() + getExtension(file.getOriginalFilename());
    Files.createDirectories(Paths.get(uploadDir));
    Files.copy(file.getInputStream(), Paths.get(uploadDir).resolve(filename));
    return "/uploads/" + filename;
}
```
